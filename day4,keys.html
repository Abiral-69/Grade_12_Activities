Keys in DBMS

Day 4

  • Key:

    A "key" refers to a special attribute or set of attributes that uniquely identify a tuple (row) within a table. Keys play a fundamental role in relational database design and management, ensuring data integrity and facilitating efficient data retrieval.

    Sample Table:

    ID Name Course Citizenship No.
    1 Abiral Computer Science 494910
    2 Aayush Economics 101049
    3 Ram Sports Science 181890
    4 Hari Biology 301289

    • Types of Keys

  • Super Key:

    The set of all possible keys that can define a tuple uniquely. In this table, all the four headers are eligible to become super key.

  • Candidate Keys:

    Keys that are eligible to become primary key and are unique in nature are called candidate keys. In the example above, all the four can be candidate keys. However, ID and Citizenship no. are more preferred.

  • Primary Key:

    The most unique key that helps to identify a tuple in a table is known as primary key. It makes finding specific data and editing, data management much easier. In the table above, the most suitable column for primary key is ID.

  • Alternate Key:

    All the candidate keys which fail to become the primary key are known as alternate keys.

  • Foreign Key:

    A foreign key is a column or columns of data in one table that refers to the unique data values, often the primary key data in another table. Foreign keys link together two or more tables in a relational database.
